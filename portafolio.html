MIS PROYECTOS

PRIMERA WEB EN HTML y CSS

Primera web echa con HTML y CSS, donde aprendí las bases para el desarrollo de una página web. Fue un trabajo en el curso intensivo realizado, donde era la primera parte, haciendo un "Hola, mundo" para darme soltura, y ver las herramientas necesarias para trabajar en una web con unos mínimos.

DADOS ECHOS CON FLEXBOX

Una de las primeras necesidades, era el poder colocar y mover a mi antojo los objetos que quería ordenar, sin tanta necesidad de márgenes, poder suprimir el "float", y hacer que un objeto, pase a otra posición, poder ordenar, hacer tablas adecuadamente, etc.. Sin duda alguna, entre grillas y flexbox, me quedo con el último, y de ahí este proyecto, para poder coger soltura en el tema.

SEMÁNTICA DE HTML5 CON FLEXBOX Y GRILLAS

Después de observar mi primer proyecto, parece que me quedé atrás hablando de HTML5, así que decidí actualizarme y coger bien las nuevas etiquetas, para una organización, optimización y que las arañas puedan rastrear mejor mi código, aprendí bien la semántica. Con este proyecto podemos ver la organización de HTML5, ordenado con flexbox y sistema de grillas. Aparte, tengo en cuenta los deben de estar sobretodo al comienzo, si juntamos varios, por ejemplo: h1 + h2 = hgroup

PÁGINA DE REGISTRO, INPUTS, RADIOS CON JavaScript

Cuando acabé la primera web en HTML y CSS, el cliente pidió una página de registro, para poder aligerar el procedimiento. Agregar botones input, y poniendo con JS, los requisitos mínimos para este registro. Por ejemplo, que el número de teléfono, sean números y tenga el mínimo de número que contienen, o que el e-mail introducido, contenga un @.. y acabe en .com .co .es .net, y así impedir a gente que pretenda poner datos irreales. También poner un radio, que tiene como misión, cancelar dos elecciones radio, al ser esta pulsada, ya que no es necesario pedir la talla de la ropa, si ellos mismos ya tienen la equipación. Por último, el saber crear una API de googleMaps, e introducir el iframe.

WEB RESPONSIVE

Tras ir acabando los primeros proyectos,lo correcto era aprender a hacerlo también para un móbil, o tablet, haciendo así mi primera web responsive. Con algún que otro detalle como un chat interno ( sin estar en una base de datos 'no se guardan los mensajes solo se muestran tras escribirlo' ). Alguna acción con JS al dar click sobre algún boton, agregar un carrusell de fotos, y hacer unos estilos más correctos, serios y dinámicos.

SINGLE PAGE

Al ver el resultado en un teléfono, y en diferentes resoluciones, descubrí algo muy interesante, y que está muy de moda para páginas con poco contenido, el cual no necesitaria hacer scroll en pantalla, llamado "Single Page", una manera de introducir todo en el mismo html y que sean separadas entre sí, con indicaciones sencillas, teniendo solo que deslizarte hacia abajo o hacia los lados para poder ver todo el contenido. Después de este proyecto aprendí que son y que pueden hacer las "media querys", algo que me hubiera ayudado mucho para poder dejar más afinado este proyecto.
